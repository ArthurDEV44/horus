# SEO Content Expert - Workflows

Processus detailles et scenarios d'utilisation pour l'agent seo-content-expert.

---

## Workflow 1 : Creation de Brief SEO

### Contexte d'utilisation

Utiliser ce workflow lorsqu'un nouveau contenu doit etre cree et qu'un brief structure est necessaire pour guider la redaction.

### Processus detaille

#### Etape 1 : Collecte des informations initiales

**Actions** :
1. Identifier le mot-cle principal cible
2. Clarifier le type de contenu souhaite (article, guide, landing page)
3. Definir l'audience cible et son niveau de maturite
4. Comprendre les objectifs business du contenu

**Questions a poser** :
- Quel est le mot-cle ou sujet principal ?
- Quelle est l'intention de recherche visee ?
- Qui est le lecteur cible ?
- Quel est l'objectif du contenu (trafic, conversion, autorite) ?

#### Etape 2 : Recherche et analyse

**Actions** :
1. Effectuer une recherche web sur le sujet pour identifier les tendances actuelles
2. Analyser les resultats de recherche pour le mot-cle principal
3. Identifier les questions People Also Ask
4. Cartographier les entites et termes semantiques pertinents
5. Evaluer la concurrence (structure, longueur, angles)

**Livrables intermediaires** :
- Liste de mots-cles secondaires
- Questions frequentes a adresser
- Analyse des gaps de contenu
- Benchmark des contenus performants

#### Etape 3 : Structuration du brief

**Elements a inclure** :

1. **Informations de contexte**
   - Mot-cle principal et volume estime
   - Intention de recherche (informationnel/commercial/transactionnel)
   - Persona cible
   - Objectif du contenu

2. **Directives SEO**
   - Mots-cles secondaires a integrer
   - Entites et termes NLP importants
   - Longueur recommandee (basee sur la concurrence)
   - Densite et placement des mots-cles

3. **Structure de contenu**
   - H1 propose (avec mot-cle principal)
   - H2 et H3 recommandes
   - Points cles a couvrir par section
   - Questions a repondre

4. **Elements E-E-A-T**
   - Signaux d'experience requis
   - Sources et donnees a citer
   - Elements d'expertise a demontrer
   - Preuves de fiabilite a inclure

5. **Directives techniques**
   - Format du title tag
   - Consignes meta description
   - Liens internes suggeres
   - Schema markup recommande

6. **Differenciateurs**
   - Angle unique a adopter
   - Valeur ajoutee par rapport a la concurrence
   - Elements visuels suggeres

#### Etape 4 : Validation et livraison

**Actions** :
1. Verifier la coherence du brief avec les objectifs
2. S'assurer que tous les elements sont actionnables
3. Presenter le brief pour validation

**Criteres de qualite** :
- Brief actionnable sans ambiguite
- Consignes SEO claires et precises
- Structure logique et complete
- Differenciateurs identifies

## Workflow 2 : Audit de Contenu Existant

### Contexte d'utilisation

Utiliser ce workflow pour evaluer et ameliorer un contenu existant qui sous-performe ou necessite une mise a jour.

### Processus detaille

#### Etape 1 : Analyse du contenu actuel

**Actions** :
1. Lire et analyser le contenu existant
2. Identifier le mot-cle principal cible
3. Evaluer l'intention de recherche adressee
4. Examiner la structure actuelle (titres, paragraphes)

**Points d'evaluation** :
- Clarte de l'objectif du contenu
- Adequation avec l'intention de recherche
- Qualite et profondeur de l'information
- Lisibilite et structure

#### Etape 2 : Analyse semantique

**Actions** :
1. Rechercher les tendances actuelles sur le sujet
2. Identifier les termes et entites manquants
3. Comparer avec les contenus top SERP actuels
4. Evaluer la couverture des questions frequentes

**Livrables** :
- Liste des gaps semantiques
- Termes NLP a ajouter
- Questions non adressees
- Sujets connexes a integrer

#### Etape 3 : Evaluation E-E-A-T

**Criteres d'evaluation** :

| Signal | Questions d'evaluation |
|--------|----------------------|
| Experience | Des exemples concrets sont-ils presents ? L'auteur demontre-t-il une connaissance pratique ? |
| Expertise | Les credentials de l'auteur sont-ils visibles ? Le niveau technique est-il approprie ? |
| Authority | Des sources fiables sont-elles citees ? Le contenu est-il cite ailleurs ? |
| Trust | Les informations sont-elles exactes et verifiables ? La date de mise a jour est-elle affichee ? |

**Livrables** :
- Score E-E-A-T par dimension
- Recommandations d'amelioration specifiques

#### Etape 4 : Audit technique on-page

**Elements a verifier** :
- Title tag (longueur, mot-cle, attractivite)
- Meta description (longueur, CTA, pertinence)
- Structure Hn (hierarchie, mots-cles)
- Alt text des images
- Maillage interne
- URL structure
- Schema markup

**Livrables** :
- Liste des problemes techniques
- Recommandations correctives

#### Etape 5 : Rapport et recommandations

**Structure du rapport** :

1. **Resume executif**
   - Score global du contenu
   - Points forts identifies
   - Axes d'amelioration prioritaires

2. **Analyse detaillee**
   - Evaluation semantique
   - Evaluation E-E-A-T
   - Evaluation technique

3. **Recommandations priorisees**
   - Actions rapides (quick wins)
   - Ameliorations structurelles
   - Optimisations avancees

4. **Plan d'action**
   - Liste des modifications a effectuer
   - Elements a ajouter
   - Contenu a supprimer ou reviser

## Workflow 3 : Optimisation pour Featured Snippets

### Contexte d'utilisation

Utiliser ce workflow pour optimiser un contenu existant afin de capturer la position zero ou les AI Overviews.

### Processus detaille

#### Etape 1 : Identification des opportunites

**Actions** :
1. Rechercher le mot-cle cible et analyser les SERP actuelles
2. Identifier le type de featured snippet present (paragraphe, liste, tableau)
3. Evaluer la difficulte de capture
4. Verifier la presence d'AI Overviews

**Types de snippets** :
- **Paragraphe** : Definitions, explications courtes
- **Liste** : Etapes, classements, enumerations
- **Tableau** : Comparaisons, donnees structurees
- **Video** : Tutoriels, demonstrations

#### Etape 2 : Analyse du snippet actuel

**Actions** :
1. Examiner le contenu actuellement en snippet
2. Identifier pourquoi Google l'a selectionne
3. Evaluer les lacunes a exploiter
4. Comprendre le format optimal

**Points d'analyse** :
- Longueur du snippet actuel
- Structure utilisee
- Qualite de la reponse
- Opportunites d'amelioration

#### Etape 3 : Optimisation du contenu

**Pour les snippets paragraphe** :
- Creer une reponse directe de 40-60 mots
- Placer immediatement apres le H2 correspondant a la question
- Commencer par une definition claire
- Utiliser un langage simple et direct

**Pour les snippets liste** :
- Structurer avec des H3 numerotes ou a puces
- Limiter a 5-8 elements
- Commencer chaque item par un verbe d'action ou terme cle
- Garder chaque item concis

**Pour les snippets tableau** :
- Utiliser des tableaux HTML propres
- Inclure des en-tetes de colonnes clairs
- Limiter les colonnes (3-4 max)
- Donnees comparables et structurees

#### Etape 4 : Optimisation pour AI Overviews

**Actions** :
1. Structurer le contenu avec des definitions claires
2. Fournir des informations factuelles et verifiables
3. Demontrer l'expertise avec des donnees uniques
4. Maintenir le contenu a jour

**Best practices** :
- Repondre directement aux questions dans les premiers paragraphes
- Utiliser des sous-titres interrogatifs (H2/H3)
- Inclure des donnees chiffrees et sources
- Couvrir les aspects connexes du sujet

#### Etape 5 : Validation et suivi

**Actions** :
1. Verifier que les optimisations sont en place
2. Suivre l'evolution des SERP apres indexation
3. Ajuster si necessaire

## Workflow 4 : Creation de Topic Cluster

### Contexte d'utilisation

Utiliser ce workflow pour structurer une strategie de contenu autour d'un sujet principal avec des contenus satellites interconnectes.

### Processus detaille

#### Etape 1 : Definition du sujet pilier

**Actions** :
1. Identifier le sujet principal a dominer
2. Evaluer le potentiel de trafic et la concurrence
3. Definir l'angle et le positionnement
4. Cartographier les sous-themes

**Criteres du sujet pilier** :
- Volume de recherche significatif
- Aligne avec l'expertise du site
- Decomposable en sous-sujets
- Pertinent pour l'audience cible

#### Etape 2 : Cartographie des sous-themes

**Actions** :
1. Rechercher les requetes associees au sujet principal
2. Identifier les questions People Also Ask
3. Analyser les sujets couverts par les concurrents
4. Regrouper par intention et angle

**Structure de mapping** :
- Sujet pilier central
- 5-10 sous-themes principaux (pages cluster)
- Questions specifiques par sous-theme
- Mots-cles par page

#### Etape 3 : Architecture de maillage

**Principes** :
- La page pilier lie vers toutes les pages cluster
- Chaque page cluster lie vers la page pilier
- Les pages cluster peuvent se lier entre elles si pertinent
- Ancres de liens descriptives et variees

**Structure de liens** :
```
        [Page Pilier]
       /   |   |   \
      /    |   |    \
  [C1]  [C2] [C3]  [C4]
    \    /     \    /
     [C1-C2]   [C3-C4]
```

#### Etape 4 : Planification editoriale

**Pour chaque page, definir** :
1. Mot-cle principal et secondaires
2. Intention de recherche
3. Angle et differenciation
4. Structure proposee
5. Priorite de creation

**Sequence recommandee** :
1. Creer la page pilier (contenu le plus complet)
2. Developper les pages cluster par priorite
3. Ajouter les liens au fur et a mesure
4. Optimiser la page pilier avec les nouveaux liens

#### Etape 5 : Maintenance et expansion

**Actions recurrentes** :
1. Mettre a jour la page pilier regulierement
2. Ajouter de nouvelles pages cluster selon les tendances
3. Renforcer le maillage interne
4. Monitorer les performances du cluster

## Workflow 5 : Revision de Contenu IA

### Contexte d'utilisation

Utiliser ce workflow pour reviser et ameliorer un contenu genere par IA afin de le rendre conforme aux guidelines Google.

### Processus detaille

#### Etape 1 : Evaluation initiale

**Actions** :
1. Lire le contenu IA dans son integralite
2. Identifier les passages generiques ou superficiels
3. Reperer les informations potentiellement incorrectes
4. Evaluer le niveau d'originalite

**Signaux d'alerte** :
- Phrases vagues ou generiques
- Absence d'exemples concrets
- Ton impersonnel et uniforme
- Information datee ou incorrecte
- Structure trop previsible

#### Etape 2 : Verification factuelle

**Actions** :
1. Verifier chaque fait et statistique
2. Valider les citations et sources
3. Mettre a jour les informations obsoletes
4. Supprimer les affirmations non verifiables

**Points de controle** :
- Dates et chiffres exacts
- Noms de personnes et organisations
- References a des etudes ou rapports
- Evolutions recentes du sujet

#### Etape 3 : Ajout de valeur humaine

**Elements a integrer** :
- Experiences personnelles et anecdotes
- Opinions et perspectives nuancees
- Exemples concrets et cas d'usage
- Insights issus de l'expertise terrain
- Nuances et exceptions au propos general

**Transformation du contenu** :
- Remplacer les generalites par des specificites
- Ajouter des "tips" issus de l'experience
- Inclure des erreurs a eviter (vecu)
- Personnaliser le ton et la voix

#### Etape 4 : Renforcement E-E-A-T

**Actions** :
1. Ajouter des citations d'experts avec sources
2. Integrer des donnees de premiere main si disponibles
3. Inclure des preuves d'experience pratique
4. Renforcer les signaux de fiabilite

**Checklist E-E-A-T** :
- [ ] Auteur identifie avec credentials
- [ ] Sources primaires citees
- [ ] Exemples de premiere main presents
- [ ] Nuances et limites exprimees
- [ ] Date de mise a jour visible

#### Etape 5 : Optimisation de la structure

**Actions** :
1. Reorganiser pour la lisibilite
2. Ameliorer les transitions entre sections
3. Varier la longueur des paragraphes
4. Ajouter des elements visuels pertinents

#### Etape 6 : Revision finale

**Checklist** :
- [ ] Le contenu apporte une valeur unique
- [ ] Le ton est authentique et humain
- [ ] Les informations sont verifiees et sourcees
- [ ] L'experience est demontree
- [ ] Le contenu respecte les guidelines Google

## Points de Decision

### Quand creer un nouveau contenu vs optimiser l'existant ?

| Situation | Decision |
|-----------|----------|
| Page existante avec trafic declinant | Optimiser l'existant |
| Nouveau mot-cle sans contenu associe | Creer un nouveau contenu |
| Contenu existant totalement obsolete | Evaluer : reecriture ou redirection |
| Gap de contenu identifie | Creer un nouveau contenu |
| Cannibalisation de mots-cles | Consolider les contenus existants |

### Quand utiliser chaque format de contenu ?

| Intention | Format recommande |
|-----------|-------------------|
| Informationnel - definition | Article court avec featured snippet |
| Informationnel - processus | Guide etape par etape |
| Informationnel - concept complexe | Guide complet long-form |
| Commercial - comparaison | Comparatif structure |
| Commercial - evaluation | Review detaille |
| Transactionnel | Landing page optimisee |

### Quand impliquer un expert externe ?

- Sujets YMYL (sante, finance, juridique)
- Domaines techniques necessitant des credentials
- Contenus necessitant des donnees proprietaires
- Sujets controverses necessitant de multiples perspectives
