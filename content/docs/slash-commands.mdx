# Slash Commands

Les **slash commands** Horus permettent de lancer des audits complets directement dans Claude Code. Chaque commande active plusieurs agents spécialisés en parallèle pour une analyse approfondie.

## Commandes disponibles

| Commande | Agents | Output |
|----------|--------|--------|
| `/horus:architecture-audit` | 10 agents | `ARCHI_OPTI_PLAN.md` |
| `/horus:seo-audit` | 7 agents | `SEO_OPTI_PLAN.md` |
| `/horus:tailwind-audit` | 8 agents | `RESPONSIVE_OPTI_PLAN.md` |

---

## Installation

### Via la Marketplace Horus (recommandé)

Dans Claude Code, exécutez ces deux commandes :

```bash
# 1. Ajouter la marketplace Horus
/plugin marketplace add https://horus-code.com/horus-marketplace/marketplace.json

# 2. Installer le plugin Horus
/plugin install horus@horus-marketplace
```

C'est tout ! Les 30 agents et les 3 slash commands sont maintenant disponibles.

### Vérification

Après installation, vérifiez que le plugin est actif :

```bash
/plugin list
```

Vous devriez voir `horus@horus-marketplace` dans la liste.

### Mise à jour

Pour mettre à jour vers la dernière version :

```bash
/plugin update horus@horus-marketplace
```

### Désinstallation

```bash
/plugin uninstall horus@horus-marketplace
```

---

## Utilisation

Dans Claude Code, tapez simplement :

```
/horus:architecture-audit
```

Claude Code va :
1. Charger la commande et activer les agents spécialisés
2. Lancer les agents en **parallèle** pour analyser votre codebase
3. Effectuer des recherches web sur les best practices officielles
4. Générer un fichier de plan d'optimisation à la racine

| `horus:coupling-expert` | Dépendances circulaires, couplage/cohésion |

## Personnalisation

### Ajouter du contexte

Vous pouvez ajouter du contexte après la commande :

```
/horus:architecture-audit

Focus particulier sur le module de paiement dans src/features/checkout/
```

## Bonnes pratiques

### Quand utiliser chaque audit

| Situation | Commande recommandée |
|-----------|---------------------|
| Nouveau projet / onboarding | `/horus:architecture-audit` |
| Avant mise en production | `/horus:seo-audit` |
| Refonte UI / nouveau design | `/horus:tailwind-audit` |
| Review trimestrielle | Les 3 audits |
| Performance dégradée | `/horus:seo-audit` (Core Web Vitals) |
| Dette technique accumulée | `/horus:architecture-audit` |

### Workflow recommandé

1. **Lancer l'audit** approprié
2. **Examiner le plan** généré (`*_OPTI_PLAN.md`)
3. **Prioriser** les tâches critiques
4. **Implémenter** par ordre de priorité
5. **Relancer l'audit** pour valider les corrections

## Configuration pour les équipes

Pour pré-configurer Horus pour toute votre équipe, ajoutez à `.claude/settings.json` :

```json
{
  "extraKnownMarketplaces": {
    "horus": {
      "source": {
        "source": "url",
        "url": "https://horus-code.com/horus-marketplace/marketplace.json"
      }
    }
  },
  "enabledPlugins": {
    "horus@horus-marketplace": true
  }
}
```

Chaque membre de l'équipe aura automatiquement accès au plugin Horus.

## Différences avec le MCP

| Aspect | Plugin (Slash Commands) | MCP |
|--------|------------------------|-----|
| **Client** | Claude Code uniquement | Claude Code, Cursor, Windsurf |
| **Installation** | `/plugin install` | Configuration MCP |
| **Mode** | Subagents parallèles | Tools individuels |
| **Output** | Fichier `.md` généré | Réponse inline |
| **Mise à jour** | `/plugin update` | Automatique |

**Recommandation** :
- **Claude Code** : Utilisez le plugin (slash commands) pour les audits complets
- **Cursor/Windsurf** : Utilisez le MCP avec le tool `activate_agent`

## Dépannage

### La commande n'apparaît pas

1. Vérifiez que le plugin est installé : `/plugin list`
2. Réinstallez si nécessaire : `/plugin install horus@horus-marketplace`
3. Redémarrez Claude Code

### Erreur "marketplace not found"

Réajoutez la marketplace :
```bash
/plugin marketplace add https://horus-code.com/horus-marketplace/marketplace.json
```

### L'audit est trop long

Les agents s'exécutent en parallèle, mais l'analyse peut prendre du temps sur de grosses codebases. Pour accélérer :
- Limitez le scope avec du contexte additionnel
- Focusez sur un dossier spécifique

### Le fichier n'est pas généré

Vérifiez que Claude Code a les permissions d'écriture à la racine du projet. Le fichier devrait apparaître après la fin de l'analyse.
